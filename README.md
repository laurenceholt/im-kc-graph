# IM KC Graph

Extracts **Knowledge Components (KCs)** from Illustrative Mathematics (IM) PDF assessment solutions using GPT vision, then serves them as a searchable static site on Netlify.

Live site: [im-kc-graph.netlify.app](https://im-kc-graph.netlify.app)

---

## For AI Agents: Project Context

This project processes Illustrative Mathematics (IM) curriculum assessment PDFs to extract granular Knowledge Components using the **KLI (Knowledge-Learning-Instruction) framework**. Each KC represents a single mathematical skill, concept, or principle that a student must apply to answer an assessment question.

### Grade Coverage

| Grade Code | Label | Units |
|-----------|-------|-------|
| GK | Kindergarten | U1-U8 |
| G1-G8 | Grade 1-8 | U1-U8 |
| G9 | Algebra 1 | U1-U8 |
| G10 | Geometry | U1-U8 |
| G11 | Algebra 2 | U1-U8 |

Assessment types: per-unit, per-section, and end-of-course.

## Architecture

```
Assessment PDFs (not in git, see .gitignore)
     |
     v
extract_questions.py    # PyMuPDF: PDFs -> question PNG images + questions.json
     |
     v
extract_kcs.py          # OpenAI GPT-5.2 vision: questions -> KCs (kcs.json)
     |
     v
dedup_kcs.py            # GPT-based within-unit deduplication
     |
     v
run_unit.py             # Orchestrates the above 3 steps for one unit
     |
     v
dedup_global.py         # Cross-unit deduplication across all units
     |
     v
site/index.html         # Static site: filters, search, question thumbnails
```

## How to Process New Units

### Step 1: Place PDFs

Put assessment solution PDFs in a folder. The pipeline accepts flexible IM filename formats:

- `IM_G5_U1_SA_Assessment.pdf` (structured: Grade 5, Unit 1, Section A)
- `Grade5-8-End-of-Course-Assessment-teacher-guide.pdf` (loose naming)

The `parse_filename()` function in `extract_questions.py` handles multiple naming patterns and maps grade codes:
- `K` -> `GK` (Kindergarten)
- `1`-`8` -> `G1`-`G8`
- `A1` -> `G9` (Algebra 1)
- `GEO` -> `G10` (Geometry)
- `A2` -> `G11` (Algebra 2)

### Step 2: Run the Pipeline

Process one unit at a time:

```bash
python run_unit.py "IM Assessments/G5" --unit G5_U1
```

**Resume flags** (useful if a step fails partway through):

```bash
# Skip question image extraction (reuse existing images)
python run_unit.py "IM Assessments/G5" --unit G5_U1 --skip-extract

# Skip KC extraction (reuse existing KCs)
python run_unit.py "IM Assessments/G5" --unit G5_U1 --skip-kcs

# Skip within-unit dedup
python run_unit.py "IM Assessments/G5" --unit G5_U1 --skip-dedup

# Combine: only redo dedup
python run_unit.py "IM Assessments/G5" --unit G5_U1 --skip-extract --skip-kcs
```

### Step 3: Global Deduplication

After processing a batch of units, run cross-unit dedup:

```bash
python dedup_global.py
python dedup_global.py
```

Run at least two passes. Stop when the number removed per pass drops to low single digits.

Preview mode (no changes written):
```bash
python dedup_global.py --dry-run
```

### Step 4: Commit and Push

```bash
git add site/data/ site/images/
git commit -m "Add G5 units, run global cross-unit dedup"
git push
```

Pushing to `main` triggers auto-deploy to Netlify.

## Pipeline Details

### extract_questions.py

- Uses **PyMuPDF (fitz)** to parse PDF pages
- Detects question boundaries using markers like "1.", "2.", "a." etc.
- Handles single-column and two-column layouts
- Outputs:
  - `site/images/{unit_id}/` — PNG images of each question
  - `site/data/{unit_id}/questions.json` — question metadata

### extract_kcs.py

- Sends question images to **GPT-5.2 vision API** (`gpt-5.2-chat-latest`)
- Batches ~5 questions per API call for efficiency
- Extracts KCs using the KLI framework (see KC Data Schema below)
- Caches raw GPT responses in `extracted_kcs/` for resume capability

### dedup_kcs.py

- Within-unit deduplication using GPT text API
- Uses GPT to find semantically similar KCs and proposes merges
- Run automatically by `run_unit.py` with `--auto` flag

### dedup_global.py

- Cross-unit deduplication across all `site/data/*/kcs.json` files
- Two-phase detection:
  1. **Same-ID detection**: KCs with identical `kc_id` across units are auto-merged
  2. **GPT fuzzy matching**: Sends all KC IDs + titles to GPT to find semantically equivalent KCs
- When merging, keeps the KC in the unit with more questions linked to it
- Caches merge groups in `extracted_kcs/global_merge_groups.json`

### run_unit.py

- Orchestrates: extract_questions -> extract_kcs -> dedup_kcs -> build_units_manifest
- The `build_units_manifest()` function rebuilds `site/data/units.json` after each run
- Exits on any step failure (non-zero exit code)

## KC Data Schema

Each KC in `kcs.json` follows the KLI framework:

```json
{
  "kc_id": "pythagorean_theorem_apply",
  "title": "Apply the Pythagorean Theorem",
  "description": "Find a missing side length in a right triangle using a^2+b^2=c^2.",
  "condition": "A right triangle with two known side lengths",
  "response": "Substitute into a^2 + b^2 = c^2 and solve for the unknown",
  "cond_type": "Variable",
  "resp_type": "Variable",
  "relationship": "Non-verbal",
  "rationale": "No",
  "label": "Principle / Model",
  "example": "Given legs 3 and 4, find hypotenuse: $3^2 + 4^2 = c^2$...",
  "notes": "",
  "question_ids": ["G8_U1_SD_a1_q2", "G8_U2_SD_a1_q1"]
}
```

KLI fields:
- **cond_type**: `Constant` (always the same) or `Variable` (changes across problems)
- **resp_type**: `Constant` or `Variable`
- **relationship**: `Verbal` (stated in words) or `Non-verbal` (inferred from context)
- **rationale**: `Yes` (requires justification) or `No`
- **label**: KC classification — `Fact`, `Concept`, `Principle / Model`, `Skill / Procedure`

## Frontend (site/index.html)

Single HTML file, no build step. Key features:
- Loads all units' data in parallel from `site/data/units.json`
- **Grade filter** — dropdown with proper labels (Kindergarten, Grade 1-8, Algebra 1, Geometry, Algebra 2)
- **Unit filter** — cascades from grade selection
- **Section filter** — filters KCs by assessment section
- **Label filter** — filters by KC type (Fact, Concept, Principle, Skill)
- **Search** — full-text search across KC titles, descriptions, and IDs
- **Question thumbnails** — click to view the original assessment question image
